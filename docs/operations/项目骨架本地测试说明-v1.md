# 项目骨架本地测试说明 v1

本文档用于指导在办公室环境中验证当前 `.NET 10` 项目骨架。

## 1. 前置条件

请先准备以下环境：

- Windows 11
- .NET 10 SDK
- SQL Server 2022
- Kepware（如需后续联调 OPC UA）

## 2. 初始化数据库

在 SQL Server 中创建平台数据库，例如：

- 数据库名：`IndustrialMonitorPlatform`

然后执行脚本：

- `deploy/sql/init/001_initial_platform_schema.sql`

该脚本会创建第一阶段最小可落地表集，并插入默认站点和默认角色。

## 3. 检查配置文件

需要重点检查以下配置文件：

- `src/IndustrialMonitor.Web/appsettings.json`
- `src/IndustrialMonitor.BridgeService/appsettings.json`

重点修改项：

- `ConnectionStrings:PlatformDb`
- `Platform:SiteCode`
- `Platform:SiteName`
- `Bridge:SourceEndpoint`

如果后续要按项目接入真实设备，还可以参考：

- `deploy/sample-config/site.template.json`
- `deploy/sample-config/devices.template.json`
- `deploy/sample-config/tags.template.json`
- `deploy/sample-config/alarms.template.json`
- `deploy/sample-config/reports.template.json`

## 4. 还原与构建

在仓库根目录执行：

```powershell
dotnet restore
dotnet build
```

如果本机只安装了 .NET 10 SDK，上述命令应按 `net10.0` 目标框架进行构建。

## 5. 启动 Web 项目

在仓库根目录执行：

```powershell
dotnet run --project .\src\IndustrialMonitor.Web\
```

启动后可验证以下地址：

- `/`
- `/health`
- `/api/platform/summary`

预期结果：

- `/health` 返回 Web 服务健康信息；
- `/api/platform/summary` 返回站点摘要占位数据；
- 首页返回简单 HTML 导航页。

## 6. 启动 BridgeService

在仓库根目录执行：

```powershell
dotnet run --project .\src\IndustrialMonitor.BridgeService\
```

预期结果：

- 控制台输出周期性 Bridge 心跳日志；
- 显示当前使用的 OPC UA 地址和轮询周期；
- 后续接入真实数据库和 OPC UA 实现后，可扩展为真实同步逻辑。

## 7. 当前版本说明

当前项目骨架属于第一版可运行结构，特点如下：

- 项目结构已按四项目拆分；
- `.NET 10` 目标框架已统一；
- Web 和 BridgeService 可独立运行；
- 当前 `Infrastructure` 里仍是占位实现，不是正式 OPC UA 和 SQL Server 实现；
- 当前重点是先确认工程结构、配置方式和启动方式。

## 8. 下一步建议

本地验证通过后，建议按以下顺序继续实现：

1. 在 `Infrastructure` 中接入真实 SQL Server 持久化；
2. 在 `Infrastructure` 中接入真实 OPC UA 客户端；
3. 在 `BridgeService` 中实现快照与历史写库；
4. 在 `Web` 中接入真实查询服务和页面；
5. 根据 Kepware 点位完成第一批真实配置。
