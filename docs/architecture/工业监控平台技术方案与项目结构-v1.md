# 工业监控平台技术方案与项目结构 v1

## 1. 目标与约束

本文档用于固化当前版本的项目技术方案，作为后续开发、部署、复制到其他控制项目时的基线。

当前已确认的前提如下：

- 现场保留现有西门子触摸屏和 WinCC，不替换现场主控制链路。
- 现场仅新增 1 台 Windows 11 mini 主机。
- 现场通过 1 台 4G 路由器提供远程访问或后续上云能力。
- 当前阶段不使用 Ubuntu 服务器。
- 平台主要面向手机和浏览器提供报表、报警、趋势、运行总览和有限授权功能。
- 项目优先考虑标准化、统一升级维护和可复制性。
- 技术栈统一采用 .NET 10。

## 2. 总体架构

整体采用“现场控制层保持不变，平台旁路增强”的思路：

```text
PLC / 西门子触摸屏 / WinCC
          |
          v
      Kepware OPC UA
          |
          v
Windows 11 mini 主机
  ├─ BridgeService（.NET 10 Worker Service）
  ├─ Web（ASP.NET Core .NET 10）
  └─ SQL Server 2022
          |
          v
   手机 / 浏览器 / 领导端
```

设计原则如下：

1. **现场控制与平台监控解耦**：PLC、触摸屏、WinCC 继续负责现场控制和本地值守。
2. **平台认单一主数据源**：如果选用 Kepware，则平台侧以 Kepware OPC UA 为主接入层。
3. **平台有自己的数据库**：平台报表、报警、审计和手机展示都读取平台库，而不是长期深度绑定 WinCC 内部数据库。
4. **项目差异配置化**：新项目尽量通过设备、点位、报警、报表配置文件切换，不靠复制改代码。

## 3. 组件职责划分

### 3.1 Kepware

职责：

- 连接 PLC。
- 管理底层驱动和扫描周期。
- 提供 OPC UA 标准访问接口。
- 处理设备通信质量码和重连。

不负责：

- 平台报表。
- 手机页面。
- 平台权限。
- 平台审计。

### 3.2 BridgeService

BridgeService 的定位不是“重新采 PLC”，而是“读取 Kepware OPC UA，完成平台侧同步和标准化”。

职责：

- 订阅或读取 OPC UA 点位。
- 将点位映射为统一业务模型。
- 写入实时快照、历史记录、同步状态。
- 生成平台级报警事件。
- 触发后续消息通知所需的数据。

不负责：

- 页面渲染。
- 用户登录。
- 复杂报表展示。

### 3.3 Web

Web 项目负责：

- 登录与权限控制。
- 报警列表、趋势、报表、设备总览。
- 手机端访问。
- 平台配置管理。
- 提供 API 给前端页面使用。

第一阶段建议：

- 使用 ASP.NET Core MVC 或 Razor Pages。
- 暂不强制引入 SignalR。
- 页面采用普通 API + 5 到 10 秒刷新策略。

### 3.4 SQL Server 2022

SQL Server 作为平台主数据库，负责：

- 设备与点位定义。
- 实时快照与历史数据。
- 报警定义与报警事件。
- 用户、角色、权限。
- 操作审计。
- 报表聚合结果。

## 4. 项目结构

推荐以一个解决方案管理多个项目，部署时仍然放在同一台 Windows 11 mini 主机上。

```text
IndustrialMonitor.Platform/
├─ IndustrialMonitor.Platform.sln
├─ src/
│  ├─ IndustrialMonitor.Web/
│  ├─ IndustrialMonitor.BridgeService/
│  ├─ IndustrialMonitor.Core/
│  └─ IndustrialMonitor.Infrastructure/
├─ tests/
│  ├─ IndustrialMonitor.Web.Tests/
│  ├─ IndustrialMonitor.BridgeService.Tests/
│  └─ IndustrialMonitor.Core.Tests/
├─ deploy/
│  ├─ windows-service/
│  ├─ sql/
│  └─ sample-config/
└─ docs/
   ├─ architecture/
   ├─ database/
   └─ operations/
```

### 4.1 IndustrialMonitor.Web

职责：

- 页面和 API。
- 权限控制。
- 报表、报警、总览展示。
- 手机浏览器访问。

建议目录：

```text
IndustrialMonitor.Web/
├─ Controllers/
├─ Pages/
├─ ViewModels/
├─ Middlewares/
├─ Services/
├─ wwwroot/
└─ Program.cs
```

### 4.2 IndustrialMonitor.BridgeService

职责：

- OPC UA 数据读取。
- 快照、历史、报警事件写库。
- 同步状态记录。
- 后续可扩展为消息通知触发源。

建议目录：

```text
IndustrialMonitor.BridgeService/
├─ Workers/
├─ OpcUa/
├─ Sync/
├─ Alarming/
├─ Persistence/
├─ Diagnostics/

```

### 4.3 IndustrialMonitor.Core

职责：

- 实体定义。
- 枚举。
- DTO。
- 平台业务服务接口。

建议目录：

```text
IndustrialMonitor.Core/
├─ Entities/
├─ Enums/

├─ Interfaces/

### 4.4 IndustrialMonitor.Infrastructure

职责：

建议目录：

IndustrialMonitor.Infrastructure/

```

## 5. 配置驱动原则

后续跨项目复制时，优先替换配置，而不是改业务代码。建议至少提供以下配置文件模板：

- `devices.json`：设备和数据源定义。
- `tags.json`：点位定义。
- `alarms.json`：平台报警规则定义。
- `reports.json`：报表定义。
- `site.json`：站点和项目基础配置。

### 5.1 devices.json 示例

```json
[
  {
    "deviceCode": "line1-plc1",
    "deviceName": "1号线PLC",
    "sourceType": "opcua",
    "endpoint": "opc.tcp://127.0.0.1:49320",
    "enabled": true
  }
]
```

### 5.2 tags.json 示例

```json
[
  {
    "tagCode": "fan01-run",
    "tagName": "1号风机运行状态",
    "deviceCode": "line1-plc1",
    "sourceNode": "ns=2;s=Channel1.Device1.Fan01.Run",
    "dataType": "bool",
    "snapshotIntervalSeconds": 5,
    "archiveEnabled": true
  }
]
```

## 6. .NET 10 基线约束

为保证统一升级和较长支持周期，项目统一采用 .NET 10，约束如下：

- Web 项目目标框架：`net10.0`
- BridgeService 目标框架：`net10.0`
- 类库项目目标框架：`net10.0`
- 优先使用 ASP.NET Core、Worker Service、EF Core 的 .NET 10 对应版本
- 不混用历史 .NET Framework 项目，除非必须对接旧组件

## 7. 第一阶段范围

第一阶段建议只交付以下内容：

- 设备总览页面
- 报警列表页面
- 趋势查询页面
- 日报或班报
- 用户与角色管理
- BridgeService 同步入库
- 设备、点位、报警配置管理

第一阶段暂不强制交付：

- 复杂远程控制
- 复杂流程组态页面
- 云端主控制链路
- 高实时推送
- 多站点集中调度

## 8. 后续演进建议

后续如需增强体验或功能，可以按顺序扩展：

1. 为报警和总览页面引入 SignalR。
2. 加入微信、短信、钉钉或邮件通知。
3. 将站点配置和点位配置从 JSON 迁移到平台配置表。
4. 引入云端只读门户和 MQTT 同步。
5. 形成多项目模板仓库。

## 9. 当前推荐定版

当前阶段的推荐定版如下：

- 现场控制：PLC + 触摸屏 + WinCC 保持不变
- 数据接入：Kepware OPC UA
- 平台同步：BridgeService（.NET 10）
- 页面与接口：ASP.NET Core Web（.NET 10）
- 数据中心：SQL Server 2022
- 部署方式：全部部署在现场 Windows 11 mini 主机
